Text Input & Preprocessing

Raw Text Processing

The first step in any NLP pipeline involves preparing raw text for machine learning models.

  • Tokenization: Breaking text into words, subwords, or characters
  • Normalization: Converting to lowercase, removing punctuation
  • Cleaning: Removing noise, HTML tags, special characters
  • Language Detection: Identifying the language of the text

Embeddings & Encoding

Converting Text to Numbers

Machine learning models require numerical input, so we convert text tokens into dense vector representations.

  • Word Embeddings: Dense vector representations that capture semantic meaning (e.g., "king" is close to "queen", "man" is close to "woman")
  • Word Embeddings Algorithms: Word2Vec (by Google), GloVe (by Stanford), FastText (by Facebook)
  • Positional Encoding: Adding position information to preserve word order
  • Attention Mechanisms: Learning relationships between different words
  • Contextualization: Understanding words based on their context

Key NLP Concepts

Tokenization Strategies

  • Word-level: Split by spaces and punctuation
  • Subword: BPE, WordPiece for handling rare words
  • Character-level: Individual characters as tokens
  • Sentence-level: Entire sentences as units

Attention Mechanisms

  • Self-attention: Words attend to other words in same sequence
  • Cross-attention: Attending between different sequences
  • Multi-head: Multiple attention patterns in parallel
  • Scaled dot-product: Efficient attention computation

Model Architectures

  • BERT: Bidirectional encoder for understanding
  • GPT: Autoregressive decoder for generation
  • T5: Text-to-text unified framework
  • RoBERTa: Robustly optimized BERT approach
